# BidEquity Newsroom — Ticket 1: Project Scaffolding & Deployment

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up a deployable Python project with a live `/healthz` endpoint, automated CI/CD to Hetzner, database migrations via Alembic, and a nightly backup to Backblaze B2 — everything needed before writing a single line of application logic.

**Architecture:** A uv-managed Python 3.12 FastAPI monolith running under systemd on a Hetzner CX22 VPS (Debian 12), reverse-proxied by Caddy which handles TLS automatically via Let's Encrypt. PostgreSQL 16 with pgvector runs on the same machine; Alembic manages schema migrations; GitHub Actions handles CI (ruff, mypy, pytest) and CD (rsync + systemd restart) on every push to `main`.

**Tech Stack:** Python 3.12, uv, FastAPI, PostgreSQL 16 + pgvector, Alembic, Caddy, systemd, GitHub Actions, Sentry, Healthchecks.io

```python
"""Sentry initialisation and Healthchecks.io ping helper."""

import logging

import httpx
import sentry_sdk

from bidequity.common.config import get_config

logger = logging.getLogger(__name__)


def init_sentry() -> None:
    """Initialise Sentry SDK. No-op if SENTRY_DSN is empty."""
    config = get_config()
    if not config.SENTRY_DSN:
        logger.info("SENTRY_DSN not set — Sentry disabled")
        return
    sentry_sdk.init(
        dsn=config.SENTRY_DSN,
        traces_sample_rate=0.1,
        profiles_sample_rate=0.1,
    )
    logger.info("Sentry initialised")


async def ping_healthcheck(slug: str, *, fail: bool = False) -> None:
    """
    Ping a Healthchecks.io check.

    Args:
        slug: The UUID or slug portion of the check URL.
        fail: If True, sends a /fail ping instead of a success ping.
    """
    config = get_config()
    if not config.HEALTHCHECKS_BASE_URL:
        return
    suffix = "/fail" if fail else ""
    url = f"{config.HEALTHCHECKS_BASE_URL.rstrip('/')}/{slug}{suffix}"
    try:
        async with httpx.AsyncClient(timeout=5) as client:
            await client.get(url)
    except Exception as exc:  # noqa: BLE001
        logger.warning("Healthchecks ping failed for %s: %s", slug, exc)
```

```python
"""Alembic environment — async mode, URL from application config."""

import asyncio
from logging.config import fileConfig

from alembic import context
from sqlalchemy.ext.asyncio import create_async_engine
from sqlmodel import SQLModel

# Import all models here so Alembic can see them.
# Add imports as new model modules are created in later tickets.
# e.g.: from bidequity.models.source import Source  # noqa: F401

from bidequity.common.config import get_config

config = context.config
if config.config_file_name is not None:
    fileConfig(config.config_file_name)

target_metadata = SQLModel.metadata


def get_url() -> str:
    return get_config().DATABASE_URL


def run_migrations_offline() -> None:
    context.configure(
        url=get_url(),
        target_metadata=target_metadata,
        literal_binds=True,
        dialect_opts={"paramstyle": "named"},
    )
    with context.begin_transaction():
        context.run_migrations()


def do_run_migrations(connection):  # type: ignore[no-untyped-def]
    context.configure(connection=connection, target_metadata=target_metadata)
    with context.begin_transaction():
        context.run_migrations()


async def run_migrations_online() -> None:
    connectable = create_async_engine(get_url())
    async with connectable.connect() as connection:
        await connection.run_sync(do_run_migrations)
    await connectable.dispose()


if context.is_offline_mode():
    run_migrations_offline()
else:
    asyncio.run(run_migrations_online())
```

### Task 9 — Nightly backup script

**Files:** `bidequity-newsroom/scripts/backup.sh`

- [ ] Create `bidequity-newsroom/scripts/backup.sh`:

```bash
#!/usr/bin/env bash
# Nightly PostgreSQL backup to Backblaze B2 via rclone.
#
# Prerequisites on VPS:
#   - rclone configured with a remote named "b2" pointing to the B2 bucket
#     (run: rclone config)
#   - B2 bucket named "bidequity-backups" created in the B2 console
#   - PGPASSWORD set in the environment (or use ~/.pgpass)
#
# Schedule: add as a systemd timer (see comments at bottom) or run from cron.
#
# Usage: ./scripts/backup.sh

set -euo pipefail

TIMESTAMP=$(date -u +"%Y-%m-%dT%H%M%SZ")
BACKUP_FILE="/tmp/bidequity-db-${TIMESTAMP}.sql.gz"
B2_REMOTE="b2:bidequity-backups/postgres"
DB_NAME="${PGDATABASE:-bidequity}"
DB_USER="${PGUSER:-bidequity}"
DB_HOST="${PGHOST:-localhost}"
RETENTION_DAYS=30

echo "[backup] Starting backup of ${DB_NAME} at ${TIMESTAMP}"

# Dump and compress
pg_dump \
  --host="${DB_HOST}" \
  --username="${DB_USER}" \
  --format=plain \
  --no-owner \
  --no-acl \
  "${DB_NAME}" \
  | gzip > "${BACKUP_FILE}"

echo "[backup] Dump complete: ${BACKUP_FILE} ($(du -h "${BACKUP_FILE}" | cut -f1))"

# Upload to B2
rclone copy "${BACKUP_FILE}" "${B2_REMOTE}/" \
  --log-level INFO

echo "[backup] Uploaded to ${B2_REMOTE}/"

# Clean up local temp file
rm -f "${BACKUP_FILE}"

# Remove remote backups older than RETENTION_DAYS
rclone delete "${B2_REMOTE}/" \
  --min-age "${RETENTION_DAYS}d" \
  --log-level INFO

echo "[backup] Pruned backups older than ${RETENTION_DAYS} days"
echo "[backup] Done"
```

## VPS Provisioning Notes (one-time manual steps)

These steps are performed once on the Hetzner CX22 and are not automated by the above tasks.

```bash
# 1. Create server in Hetzner Console: CX22, Debian 12, Frankfurt datacenter
#    Add your SSH public key during creation.

# 2. SSH in and harden
ssh root@YOUR_VPS_IP
apt update && apt upgrade -y
useradd --system --create-home --shell /bin/bash bidequity
mkdir -p /opt/bidequity-newsroom
chown bidequity:bidequity /opt/bidequity-newsroom

# 3. Install PostgreSQL 16 + pgvector
apt install -y postgresql-16 postgresql-16-pgvector
sudo -u postgres createuser --pwprompt bidequity
sudo -u postgres createdb --owner=bidequity bidequity
sudo -u postgres psql -c "CREATE EXTENSION IF NOT EXISTS vector;" bidequity

# 4. Install Caddy
apt install -y debian-keyring debian-archive-keyring apt-transport-https
curl -1sLf 'https://dl.cloudflare.com/cloudflare-gpg/gpg.key' \
  | gpg --dearmor -o /usr/share/keyrings/cloudflare-main.gpg
curl -1sLf 'https://dl.cloudflare.com/cloudflare-gpg/public.key' \
  | gpg --dearmor -o /usr/share/keyrings/cloudflare-main.gpg
apt install -y caddy

# 5. Install uv
curl -LsSf https://astral.sh/uv/install.sh | sh

# 6. Install rclone for backup
curl https://rclone.org/install.sh | bash
# Then: rclone config  (set up b2 remote)

# 7. Create secrets directory
mkdir -p /etc/bidequity
chmod 700 /etc/bidequity
# Copy .env.example → /etc/bidequity/.env and fill in real values

# 8. Allow bidequity user to restart its own service
echo "bidequity ALL=(root) NOPASSWD: /bin/systemctl restart bidequity-newsroom" \
  > /etc/sudoers.d/bidequity

# 9. Deploy initial code
# (This happens via GitHub Actions after repo is pushed)
# For first-time manual deploy:
cd /opt/bidequity-newsroom
uv sync --no-dev
uv run alembic upgrade head

# 10. Install and start services
cp bidequity-newsroom.service /etc/systemd/system/
cp bidequity-backup.service /etc/systemd/system/
cp bidequity-backup.timer /etc/systemd/system/
systemctl daemon-reload
systemctl enable --now bidequity-newsroom
systemctl enable --now bidequity-backup.timer

# 11. Configure Caddy
cp Caddyfile /etc/caddy/Caddyfile
# Edit /etc/caddy/Caddyfile with real domain
# Set DASHBOARD_USERNAME + DASHBOARD_PASSWORD_HASH in Caddy's environment
systemctl restart caddy
```

Response Requirements view is a reference depository, NOT a tracking view. Shows what the client released for submission.

**Why:** The bid manager's first job after ITT receipt is to understand the shape, scale, and structure of what's been asked. Production tracking belongs on the Production Pipeline tab.

**How to apply:**

Metrics bar:
- Total number of questions split by category (quality, price, social value, etc.) — not just a section count
- Hurdle questions flagged and surfaced
- Effort indicator (influenced by page/word limits in qualitative section)
- Do NOT show: total marks, production started, progress %, compliance

Register table:
- Reference with document location (section X, paragraph Y) so bid manager can trace back to source
- Deliverable title
- Words (output volume — word/page limits)
- Individual question marks/score (NOT section weight %). Clarification: section gets a % of total (e.g. quality = 60%), then each question within that section has marks that add up to 100% within the section. It's the per-question score that matters.
- Do NOT show: status, effort, owner, action/status dropdown

Sort order:
- Follow client's prescribed document structure (company info → quality → pricing → financial → social value → other)
- Do NOT sort by marks or priority — mirror the ITT structure

Slideout:
- Question text, evaluation criteria, response type, section reference, max score, hurdle, word/page limits — all reference data
- Remove ALL production data (status, owner, word count progress, complexity, effort, transitions, quality dimensions, notes)

Pipeline status bar:
- Remove from this view entirely — belongs on Production Pipeline tab only

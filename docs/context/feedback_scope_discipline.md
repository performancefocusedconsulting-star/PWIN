Keep the product focused. User is actively worried about scope creep turning this into a god-project.

**Why:** The product isn't finished yet. Adding ambitious new entity clusters (requirements traceability, clarification management) before the core is battle-tested risks architecture blowout. AI capability overlay is still to come.

**How to apply:** When proposing new entities, views, or features, always ask: can this be managed outside the app and ingested later? Prefer deferral to V2 over building speculative structure. Don't add new Submissions module views unless existing ones are working and tested. Build data targets for AI skills, not the skills themselves.
